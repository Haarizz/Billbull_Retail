Hilite Startup Issue (Resolved)

During migration to systemd, the Hilite service failed because Flyway migration V6__security_unique_email.sql attempted to create a unique index on active user emails. The database contained two active users with the same email address.

UPDATE users
SET is_active = false
WHERE id = 1;

systemctl restart billbull-hilite

Root Cause: Duplicate active email in the users table, not a systemd or Playwright issue.
