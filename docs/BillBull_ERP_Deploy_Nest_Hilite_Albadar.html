1 · Client Map (Ports & Databases)

Client Domain Backend Port Database Spring Profile
Nest nest.billbull.app 8080 billbull_nest nest
Hilite hilite.billbull.app 8081 billbull_hilite hilite
Albadar albadar.billbull.app 8082 billbull_albadar albadar
Helenz helenz.billbull.app 8083 billbull_helenz helenz
QA qa.billbull.app 8084 billbull_qa qa
Architecture recap: One shared React build served by Nginx · Five independent Spring Boot JAR instances (one per client) · One PostgreSQL server with five separate databases · Nginx routes each subdomain to the correct backend port.

3 · Database Setup

Step 1 — Connect to PostgreSQL

BASH
sudo -u postgres psql

Step 2 — Create Databases and User

Run inside psql:

SQL
CREATE USER billbull WITH PASSWORD 'root';

CREATE DATABASE billbull_nest;
CREATE DATABASE billbull_hilite;
CREATE DATABASE billbull_albadar;
CREATE DATABASE billbull_helenz;
CREATE DATABASE billbull_qa;

GRANT ALL PRIVILEGES ON DATABASE billbull_nest     TO billbull;
GRANT ALL PRIVILEGES ON DATABASE billbull_hilite   TO billbull;
GRANT ALL PRIVILEGES ON DATABASE billbull_albadar  TO billbull;
GRANT ALL PRIVILEGES ON DATABASE billbull_helenz   TO billbull;
GRANT ALL PRIVILEGES ON DATABASE billbull_qa       TO billbull;

\l
\q

Verify: All 5 databases listed. Each owned by postgres with billbull having privileges.

WorkingDirectory=/root/billbull/billbull-backend
ExecStart=/usr/bin/java -jar target/billbull-backend-0.0.1-SNAPSHOT.jar --spring.profiles.active=nest
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]

7 · Nginx Reverse Proxy

Step 1 — Create Temporary SSL Options File

BASH
mkdir -p /etc/letsencrypt

cat > /etc/letsencrypt/options-ssl-nginx.conf << 'EOF'
ssl_protocols TLSv1.2 TLSv1.3;
ssl_prefer_server_ciphers on;
ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384';
ssl_session_cache shared:le_nginx_SSL:10m;
ssl_session_timeout 1440m;
ssl_session_tickets off;
EOF

openssl dhparam -out /etc/letsencrypt/ssl-dhparams.pem 2048

Step 2 — Create Nginx Config

BASH
cat > /etc/nginx/sites-available/billbull << 'EOF'
# HTTP → HTTPS redirect for all clients
server {
    listen 80;
    server_name nest.billbull.app hilite.billbull.app albadar.billbull.app helenz.billbull.app qa.billbull.app;
    return 301 https://$host$request_uri;
}

# ── NEST (port 8080) ─────────────────────────────────────────
server {
    listen 443 ssl;
    server_name nest.billbull.app;
    root /var/www/billbull;
    index index.html;
    client_max_body_size 300M;

    ssl_certificate     /etc/letsencrypt/live/nest.billbull.app/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/nest.billbull.app/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    location / { try_files $uri $uri/ /index.html; }
    location /assets/ { expires 1y; add_header Cache-Control "public, immutable"; }
    location /uploads/ { proxy_pass http://127.0.0.1:8080/uploads/; }
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Authorization $http_authorization;
    }
}

13 · Troubleshooting

Backend Won't Start

BASH
journalctl -u billbull-nest -n 100 --no-pager
  • DB connection failed → Check PostgreSQL is running: systemctl status postgresql
  • Port already in use → Change port in the matching application-*.properties
  • Flyway migration error → Ensure spring.flyway.enabled=false is present
  • JAR not found → Re-run mvn clean package -DskipTests

Nginx 502 Bad Gateway

Backend not responding on that port.

BASH
# Example for nest (port 8080)
systemctl status billbull-nest
curl -v http://127.0.0.1:8080/api/auth/login

SSL Certificate Not Found

BASH
ls -la /etc/letsencrypt/live/
# Re-run certbot for missing domains
certbot certonly --standalone -d missing.billbull.app

Password Not Working After Update

Backend cached user in memory. Fix: restart the service.

BASH
systemctl restart billbull-nest billbull-hilite billbull-albadar billbull-helenz billbull-qa

Daily Health Check

BASH
# All services
systemctl status billbull-nest billbull-hilite billbull-albadar billbull-helenz billbull-qa

# Recent errors per service
